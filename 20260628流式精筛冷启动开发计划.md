## 二、核心架构变更

### 2.1 分层路由重定义

```
Layer 0: 硬筛 (纯代码, ~30s)
    │
    ▼ ~4500 只
Layer 1: M1/M3 批量粗筛 (5 级分档, ~1h)
    │
    ├── 强烈推荐 (~10 只) ──→ whitelist ──→ Layer 3 (立即)
    ├── 推荐     (~750 只) ──→ Layer 2 流式堆
    ├── 中性     (~2000 只) ──→ ✗ 放弃 (本次不考虑)
    ├── 回避     (~1000 只) ──→ ✗ 放弃 (本次不考虑)
    └── 卖出     (~740 只)  ──→ blacklist (120 天)
    
Layer 2: DSV4-Pro 流式双堆 top-α (~3min, 与 Layer 1 重叠)
    │
    └── top-α dispatch → Layer 3
    
Layer 3: 7Agent+3Scorer (semaphore=5, 批次 20, ~1.8h)
    │
    └── 分数截断 1:1.2 → 精筛池
```

### 2.2 关键设计决策

| 决策 | 内容 | 理由 |
|:---|:---|:---|
| 推荐直接进 Layer 2 | "推荐"不再去 Layer 2 排队等 Layer 1 完成 | M1/M3 已确认有投资价值, 不需要 M2 二次审核 |
| 中性/回避放弃 | 不进 Layer 2, 本次不再考虑 | 30 天全量更新时会重新分类 |
| Layer 2 换 DSV4Pro | 排序任务需要 thinking 深度 | 750 只需 ~3min, 不占关键路径 |
| 流式双堆 top-α | τ 稳定后才 dispatch | 避免过早 dispatch 导致 Layer 3 浪费 |
| Layer 3 并发 3→5 | 压缩尾巴 1.3h→0.8h | 总耗时 2.3h→1.8h |
| 固定比例截断 | 淘汰末尾 0.2/1.2 | 比 LLM 动态阈值更可预测 |

### 3.3 Layer 2: 流式双堆 top-α

#### 3.3.1 新增类: `StreamingTopAlphaHeap`

```python
class StreamingTopAlphaHeap:
    """
    流式双堆 top-α 选择器。
    
    维护:
      - High: 小顶堆, 存当前 top-α 候选
      - Low:  大顶堆, 存其余
    
    τ = High.min() = 当前 top-α 下界阈值
    
    Dispatch 策略:
      连续 stable_n 批 τ 波动 < ε → τ 收敛 → dispatch High 中未发股票
    """
    
    def __init__(self, alpha_fn: Callable[[], float], 
                 epsilon: float = 3.0, stable_batches: int = 3):
        self.alpha_fn = alpha_fn          # 动态计算 α
        self.epsilon = epsilon            # τ 稳定阈值
        self.stable_batches = stable_batches
        self.high = []                    # min-heap: (score, stock_id)
        self.low = []                     # max-heap: (-score, stock_id)
        self.dispatched = set()           # 已 dispatch 的 stock_id
        self.tau_history = []             # 最近 N 批的 τ 值
        self.total = 0                    # 总样本数
    
    @property
    def tau(self) -> float:
        """当前 top-α 阈值"""
        return self.high[0][0] if self.high else 0.0
    
    def alpha(self) -> float:
        """当前 α 值 (由外部闭包动态计算)"""
        return self.alpha_fn()
    
    def feed(self, score: float, stock_id: str):
        """喂入一个新分数"""
        self.total += 1
        
        # Step 1: 粗分
        if not self.high or score >= self.high[0][0]:
            heapq.heappush(self.high, (score, stock_id))
        else:
            heapq.heappush(self.low, (-score, stock_id))
        
        # Step 2: 按比例重平衡
        k = max(1, math.ceil(self.alpha() * self.total))
        while len(self.high) > k:
            s, sid = heapq.heappop(self.high)
            heapq.heappush(self.low, (-s, sid))
        while len(self.high) < k:
            s, sid = heapq.heappop(self.low)
            heapq.heappush(self.high, (-s, sid))
        
        # Step 3: 修正跨堆次序
        while self.low and self.high and -self.low[0][0] > self.high[0][0]:
            a_s, a_id = heapq.heappop(self.low)
            b_s, b_id = heapq.heappop(self.high)
            heapq.heappush(self.high, (-a_s, a_id))
            heapq.heappush(self.low, (-b_s, b_id))
    
    def batch_done(self):
        """一批喂完后调用, 记录 τ 历史"""
        self.tau_history.append(self.tau)
        if len(self.tau_history) > self.stable_batches * 2:
            self.tau_history = self.tau_history[-self.stable_batches * 2:]
    
    def is_tau_stable(self) -> bool:
        """检查 τ 是否连续 stable_batches 批稳定"""
        if len(self.tau_history) < self.stable_batches:
            return False
        recent = self.tau_history[-self.stable_batches:]
        return max(recent) - min(recent) < self.epsilon
    
    def should_dispatch(self) -> bool:
        return self.is_tau_stable()
    
    def get_undispatched(self) -> List[str]:
        """获取 High 中尚未 dispatch 的 stock_id"""
        result = []
        for _, sid in self.high:
            if sid not in self.dispatched:
                result.append(sid)
                self.dispatched.add(sid)
        return result
    
    def finalize(self) -> List[str]:
        """Layer 1 完成后调用, dispatch High 中剩余所有"""
        return self.get_undispatched()  # 此时 High 已最终确定
```

#### 3.3.2 α 动态计算

```python
def calc_alpha(whitelist_count: int, recommended_total: int, 
               target_size: int, ratio: float = 1.2) -> float:
    """
    α = (target_size × ratio - whitelist_count) / recommended_total
    
    示例:
      target_size=100, ratio=1.2, whitelist=10, recommended=750
      α = (120 - 10) / 750 = 0.147 → 15%
    """
    layer3_capacity = int(target_size * ratio)
    layer2_slots = max(0, layer3_capacity - whitelist_count)
    if recommended_total == 0:
        return 0.0
    return min(1.0, layer2_slots / recommended_total)
```

#### 3.3.3 τ 收敛预估

```
n=50  (前 30s):   SE(τ) ≈ 5.3 分 → 剧烈波动, 不 dispatch
n=100 (前 60s):   SE(τ) ≈ 3.75 → 接近阈值
n=200 (前 120s):  SE(τ) ≈ 2.65 → 连续 3 批 <3 分 → 开始 dispatch
n=500 (前 180s):  SE(τ) ≈ 1.68 → 锁定, 持续 dispatch
```

### 3.5 总调度器: `run_pool_update` 重写

```
async def run_pool_update(term, on_stage, on_progress):
    # ── Layer 0: 硬筛 (~30s) ──
    ts_stocks = await hard_screen()
    
    # ── 初始化共享状态 ──
    whitelist = []
    blacklist = []
    layer3_queue = asyncio.Queue(maxsize=200)
    layer3_results = []
    scored_candidates = []
    candidates_lock = asyncio.Lock()
    
    target_size = {"short": 100, "medium": 80, "long": 60}[term]
    ratio = 1.2
    
    # Layer 2 流式堆
    def alpha_fn():
        return calc_alpha(len(whitelist), len(recommended_score_map), target_size, ratio)
    stream_heap = StreamingTopAlphaHeap(alpha_fn=alpha_fn)
    recommended_score_map = {}  # {stock_id: layer2_score}
    
    # Layer 3 消费者启动
    layer3_task = asyncio.create_task(
        _layer3_consumer(layer3_queue, layer3_results, candidates_lock, term, on_progress)
    )
    
    # ── Layer 1: 流式批量粗筛 (边跑边分叉) ──
    async for batch in batch_score_layer1_stream(ts_stocks, term):
        new_recommended = []
        
        for stock in batch:
            match stock.layer1_level:
                case "强烈推荐":
                    whitelist.append(stock)
                    await layer3_queue.put(stock)
                case "推荐":
                    new_recommended.append(stock)
                case "卖出":
                    blacklist.append(stock)
                case _:  # 中性/回避
                    pass
        
        # 新推荐 → Layer 2 打分 → 喂流式堆
        if new_recommended:
            layer2_scores = await score_layer2_batch(
                new_recommended, term, model_suffix="_6"
            )
            for stock, score in zip(new_recommended, layer2_scores):
                recommended_score_map[stock.code] = score
                stream_heap.feed(score, stock.code)
            
            stream_heap.batch_done()
            
            # τ 稳定 → dispatch
            if stream_heap.should_dispatch():
                for code in stream_heap.get_undispatched():
                    stock = code_to_stock[code]
                    await layer3_queue.put(stock)
        
        if on_progress:
            on_progress(...)
    
    # Layer 1 完成 → Layer 2 finalize
    stream_heap.finalize()
    for code in stream_heap.get_undispatched():
        await layer3_queue.put(code_to_stock[code])
    
    # 标记队列完成
    layer3_done_event.set()
    await layer3_queue.join()
    
    # ── 截断到 target_size ──
    async with candidates_lock:
        scored_candidates.sort(key=lambda x: x.final_score, reverse=True)
        pool = scored_candidates[:target_size]
    
    # ── 持久化 ──
    pm = PoolManager()
    pm.clean_expired_blacklist()
    pm.update_pool(term, pool)
    
    # 黑名单入库
    for b in blacklist:
        pm.add_to_blacklist(b.code, b.layer1_reason, 120)
    
    return {"term": term, "pool": pool, "whitelist": whitelist, 
            "blacklist": blacklist, "final_pool_size": len(pool)}
```

## 四、时间线

```
t=0         Layer 0 ─┤ 30s
t=30s       Layer 1 第一批 ─────────────────┤ 1h
t=30s+4s    第一批 5 只完成, 路由分发
t=2min      Layer 2 τ 波动, 喂入 50 只推荐
t=3min      Layer 2 τ 收敛 → 首批 ~50 只 dispatch → Layer 3 开跑
            Layer 3 semaphore=5, 第一批 5 只并发开始
t=5min      Layer 3 第 1-2 批完成, 继续消费队列
t=1h        Layer 1 完成, Layer 2 完成
            Layer 3 已运行 57min → 已处理 ~85 只
            剩余 ~35 只
t=1h+28min  Layer 3 剩余 35 只 / 5并发 / 200s = 1400s ≈ 23min
            但因为并发批次间的 IO 等待, 实际 ~28min
t≈1.5h     Layer 3 全部完成
            排序截断 → 精筛池写入
            总耗时 ≈ 1.5h
```

**关键路径**: `Layer 0(30s) + Layer 3(1.5h) = ~1.6h`
**保守估计**: API 波动 + 重试 → **~1.8h**
