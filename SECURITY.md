# 安全政策

## 支持的版本

本项目目前处于活跃开发阶段，仅支持最新的主分支版本。

| 版本 | 支持状态 |
|------|----------|
| main 分支 | ✅ 支持 |
| 历史版本 | ❌ 不支持 |

## 报告安全漏洞

如果你发现安全漏洞，请通过以下方式报告：

### 方式一：GitHub Security Advisory（推荐）

1. 访问项目仓库的 **Security** 标签页
2. 点击 **Report a vulnerability**
3. 填写漏洞详情

### 方式二：GitHub Issue

如果漏洞不包含敏感信息，可以直接创建 Issue 并标记为安全相关。

**注意**：对于严重的安全漏洞，请优先使用 Security Advisory 以避免公开披露。

## 安全最佳实践

### API Key 管理

- **永远不要**将 API Key 提交到代码仓库
- 使用 `.env` 文件管理敏感配置
- 确保 `.env` 在 `.gitignore` 中
- 定期检查是否有意外泄露

### 数据保护

- 本系统处理的金融数据存储在用户本地
- 不向第三方传输用户数据
- LLM API 调用可能涉及股票分析数据，请遵守相关数据源的使用条款

### 依赖安全

- 定期更新依赖以修复已知漏洞
- 使用 `pip-audit` 等工具检查依赖安全性
- 生产环境建议锁定依赖版本

## 响应时间

- 初次响应：7 个工作日内
- 漏洞确认：14 个工作日内
- 修复发布：视漏洞严重程度而定

## 致谢

我们会在修复后（征得报告者同意后）在 Release Notes 中致谢安全漏洞报告者。
