## 项目简介

本项目是一个面向 **A 股市场投资研究** 场景的开源多智能体分析平台，定位为 **AI 驱动的投研辅助工具**，旨在探索大语言模型（LLM）在金融数据分析、多维度个股/基金评估、策略回测等领域的应用潜力。

**核心定位**：本项目是一个**研究工具**，而非投资产品。它帮助研究者、学生和量化爱好者理解如何用多智能体协作框架进行结构化投研分析，并提供可复现的消融实验平台来量化各分析维度的真实贡献。

**核心优势**：
- 7 个专业化分析 Agent 并行运行，交叉验证，减少单模型偏见
- 结构化证据体系（SignalPack），每条分析结论均有数据支撑和来源标注
- 内置反幻觉设计，严格区分数据事实与模型推断
- 完整的评估控制塔（14 条模拟盘 + 23 条回测线路），用消融实验量化每个 Agent 的贡献
- 双缓存架构，热缓存命中时全链路跳过 LLM 调用

> **关于操作方式**：本系统以 **Streamlit Web UI** 为主要操作界面，所有功能均可通过浏览器中的图形界面完成。以下各功能中附带的 CLI 命令面向**开发者和运维人员**，适用于自动化脚本、定时任务(cron)、批量处理和调试场景，与 Web UI 调用的是同一套后端逻辑。**日常使用推荐通过 Web UI 操作。**

```
         └──────────────┘     └──────────────┘
```

**核心数据流**：
- 「股票查询」→ 分析结果可存入「股票池」→ 池内评分数据供「模拟分析与迭代」使用
- 「模拟分析与迭代」→ 精筛池数据 + 模拟盘结果 → 为「智能投顾」提供数据基础
- 「智能问答」→ 独立对话引擎，可查询个股数据，但与评分系统无直接依赖
- 「基金专区」→ 独立管线，与 A 股分析架构平行

**推荐使用顺序**：
1. 先体验「股票查询」，了解单股分析能力
2. 将感兴趣的股票加入「股票池」，进行打分对比
3. 如有批量需求，使用「批量打分」上传 Excel
4. 前往「模拟分析与迭代」更新精筛池（首次冷启动约 1.5~2 小时/池）
5. 精筛池更新完成后，可使用「智能投顾」的各项功能

### 功能一：股票查询（★ 推荐优先体验）

**页面入口**：侧边栏 → 「股票查询」

**功能说明**：输入任意 A 股股票代码（如 `603871`）或名称（如 `嘉友国际`），系统启动 7 个专业化分析 Agent 并行分析，产出完整的个股深度研究报告。

**两种使用模式**：

| 模式 | 说明 | 耗时 |
|------|------|------|
| **快速查询** | 调用 M5 轻量模型快速回答，适合快速了解 | ~10-30 秒 |
| **深度报告** | 启动完整 7 Agent 管线，生成 9 段式报告 | ~3-5 分钟 |

**深度报告内容**（9 段式格式）：
1. 综合结论与操作建议
2. 信号总览（7 Agent 汇总）
3. 看多因素分析
4. 看空因素分析
5. 关键时间节点（财报、解禁、分红等）
6. 短/中/长线三维度判断
7. 风险提示
8. 分析置信度
9. 免责声明

### 功能二：股票池管理

**页面入口**：侧边栏 → 「股票池」

**功能说明**：管理 4 个独立股票池（快筛池 / 短线池 / 中线池 / 长线池），支持手动添加股票、触发评分、查看评分详情。

**四个池的定位**：

| 池 | 用途 | 评分维度 |
|----|------|---------|
| **快筛池** | 轻量初筛，快速分类 | M2 轻量模型快速打分 |
| **短线池** | 短线交易机会追踪 | 技术面(25) + 量价(20) + 资金(20) + 事件(20) + 舆情(15) |
| **中线池** | 中线价值投资 | 基本面(20) + 质量(20) + 估值(15) + 事件(15) + 技术(10) + 行业(10) + 舆情(10) |
| **长线池** | 长线深度价值 | 回报(25) + 质量(20) + 估值(15) + 护城河(15) + 资金(10) + 政策(10) + 技术(5) |

**操作流程**：
1. 选择目标池（快筛/短线/中线/长线）
2. 输入股票代码和名称，点击「添加」
3. 点击个股行上的「打分」按钮，触发三维度评分
4. 评分完成后可查看详细的评分报告

<details>
<summary>开发者可选 — CLI 命令（与 Web UI 功能等价）</summary>

```bash
python -m src.main_pool add 603871 嘉友国际        # 添加到中线池（默认）
python -m src.main_pool add 603871 嘉友国际 --term short  # 添加到短线池
python -m src.main_pool score 603871               # 三维度评分
python -m src.main_pool report 603871              # 查看评分详情
```

</details>

### 功能三：批量打分

**页面入口**：侧边栏 → 「批量打分」

**功能说明**：上传包含股票代码的 Excel 文件，系统自动批量调用评分管线，对全部股票进行打分并分级排序。

**操作流程**：
1. 准备 Excel 文件（需包含 `code` 和 `name` 两列）
2. 上传文件，选择评分期限（短线/中线/长线）
3. 系统自动批量打分，实时显示进度
4. 完成后按评分等级（A/B/C/D）分类展示结果
5. 支持一键导出结果到 Excel

**适用场景**：
- 大规模初筛：从数百只候选中快速找出高分标的
- 板块对比：同板块多只股票的横向评分对比
- 定期复盘：每月/每季批量更新评分

### 功能四：智能问答

**页面入口**：侧边栏 → 「智能问答」

**功能说明**：ChatGPT 风格的开放式对话界面，可以与 AI 投研分析师进行多轮自然语言交流。

**支持的问答类型**：
- 个股基本面/技术面/估值分析（如："贵州茅台的 ROE 趋势如何？"）
- 行业对比（如："白酒行业 PE 分位数处于什么位置？"）
- 市场热点追踪（如："最近哪些板块资金流入最多？"）
- 财务数据查询（如："比亚迪最近三年营收增长率是多少？"）
- 投资策略讨论（如："当前市场适合定投还是择时？"）

**与「股票查询」的区别**：
- 「股票查询」是结构化的完整报告输出，适合深度了解单只股票
- 「智能问答」是自由对话，适合快速提问、多轮追问、横向对比

### 功能五：基金专区

**页面入口**：侧边栏 → 「基金专区」

**功能说明**：面向公募基金和 ETF 的独立分析管线，采用 7 个基金专用 Agent 并行分析 + 6 维度综合评分 + 10 道基金风险门控。

**分析维度**：
- 产品文档（基金类型、基准、策略）
- 业绩风险（净值表现、回撤、风险指标）
- 持仓分析（组合构成、集中度）
- 基金经理（经验、风格、稳定性）
- 基准比较（跟踪误差、风格漂移）
- 费率评估（费率结构 vs 同类）
- 基金事件（经理变更、分红等）

### 功能六：模拟分析与迭代（⚠ 持续完善中）

**页面入口**：侧边栏 → 「模拟分析与迭代」

**功能说明**：本系统的自我迭代引擎，包含 14 条实盘模拟线路和 23 条回测线路，通过消融实验量化每个分析 Agent 的真实贡献。

> **注意**：此功能目前仍在持续优化中，部分功能可能不稳定。详细待改进事项见 `模拟分析与迭代补充开发修改指南.md`。

**三个 Tab 页**：

| Tab | 功能 | 说明 |
|-----|------|------|
| **模拟盘运营** | 日常运营面板 | 持仓查看、池健康度、一键检查（调仓+结算）、精筛池更新 |
| **回测与消融** | 研究分析 | 历史回测、Agent 贡献度归因、消融实验 |
| **趋势与优化** | 长期追踪 | 评分趋势图、Loss 趋势、贡献度排行、自动优化建议 |

**精筛池更新（重要）**：
- 使用「智能投顾」功能前**必须**先在此处全量更新精筛股票池
- 点击「更新精筛池」按钮，选择期限（短线/中线/长线），触发四层筛选管线
- 首次冷启动耗时约 **1.5~2 小时/池**，后续热缓存更新快得多
- 更新过程以独立子进程运行，关闭页面不会中断（支持跨重启存活）

```bash
# 1. 克隆仓库
git clone <your-repo-url>
```

```bash
# Linux / WSL
./run.sh start

# Windows CMD
run.bat start

# Windows PowerShell
.\run.ps1 start
```

启动后在浏览器打开 `http://localhost:8501`。

### 多模型配置

本系统使用 **6 个独立的 LLM 模型**，按任务特性分配以平衡分析质量与调用成本：

| 代号 | 默认模型 | 适用角色 |
|------|---------|---------|
| **M1** | MiMo-V2.5-Pro | 总结报告、中/长线评分、基本面、估值、质量风控、基金评分 |
| **M2** | Qwen3.6-Flash | 快速查询、快筛初筛、批量打分 |
| **M3** | Qwen3.7-Plus | 技术面、新闻、短线评分、事件、资金流向、基金子代理 |
| **M4** | (已迁移至 M1) | 保留位，避免破坏配置读取 |
| **M5** | MiMo-V2.5 | 评估系统 Agent 分析（成本优化）、智能问答 |
| **M6** | DeepSeek V4 Pro | 评估编排、归因诊断、报告撰写、LLM Free 策略 |

## 项目优势

1. **多 Agent 协作减少偏见**：7 个专业化 Agent 各司其职，交叉验证，避免单一模型的全知偏见。每个 Agent 使用最适合其任务特性的模型（如技术面用擅长模式识别的 Qwen3.7-Plus，基本面用擅长推理的 MiMo-V2.5-Pro）。

2. **结构化证据可追溯**：每条分析结论都以 SignalPack 格式存储，包含数据因子、方向、强度、来源等级，支持事后审计和冲突检测。

3. **反幻觉设计**：严格区分「数据事实区」和「分析判断区」，缺失数据使用分级语言（"无法评估"/"基于不完整数据"/"存在数据缺口"），从机制上减少 LLM 编造数据的可能。

4. **可量化的 Agent 贡献评估**：评估控制塔通过消融实验（ablation）量化每个 Agent 对最终评分的贡献度，配合 Bootstrap CI 和 Permutation Test 提供统计显著性检验，而非仅凭主观判断。

5. **双缓存架构**：生产缓存和评估缓存物理隔离，避免不同模型的输出交叉污染。热缓存命中时全链路跳过 LLM 调用，大幅降低重复分析的成本。

6. **双资产类别覆盖**：同时支持 A 股个股和公募基金/ETF 的深度分析，两条管线独立运行，架构平行。

7. **跨重启存活设计**：精筛池更新等耗时操作以独立子进程运行，关闭浏览器或重启 Streamlit 不影响后台任务，支持断点续传。

## 项目不足与改进方向

本着诚实透明的原则，以下是当前版本已知的不足之处：

1. **冷启动成本高**：首次全量更新精筛池需要约 1.5~2 小时/池，且消耗大量 LLM API 调用额度。Layer 3 的并发度受限于 MCP stdio server 的单进程瓶颈。

2. **模拟分析与迭代功能尚未完全稳定**：部分消融实验线路的统计检验样本量不足（需 ≥10 批次），趋势图表在数据积累初期无法渲染。Worker 进程异常退出时的状态恢复机制仍有改进空间。

3. **智能投顾功能仍在开发中**：AI 顾问对话、股票推荐、持仓管理等子功能的实现深度有限，部分功能依赖精筛池数据的前置更新。

4. **数据源单一性**：A 股数据主要依赖 Tushare Pro，当 Tushare API 不可用时系统无法运行（设计上不允许回退到模拟/估算数据）。国际市场数据通过 AKShare 和 Yahoo Finance 补充，但覆盖度有限。

5. **模型依赖外部 API**：6 个模型分属不同供应商（小米/阿里/DeepSeek），任何一家的 API 变更、限流或停服都可能影响系统运行。

6. **回测数据的时效性**：回测结果基于历史数据，不代表未来表现。回测线路的策略实现为纯代码规则，未考虑真实市场中的流动性冲击、停牌等极端情况的部分细节。

7. **文档和测试覆盖率有待提升**：部分模块的单元测试覆盖不够全面，API 文档尚不完整。

**我们诚挚欢迎社区贡献者参与改进**，包括但不限于：
- 优化精筛池冷启动性能（提升并发、改进缓存策略）
- 完善模拟分析与迭代的统计检验和可视化
- 增强智能投顾的对话质量和推荐算法
- 补充单元测试和集成测试
- 改善文档和用户体验
